# Discrepancy Audit — current implementation vs. Figma frames

Generated by walking every Figma frame side-by-side with a screenshot of the equivalent state in the running app. Discrepancies are categorized and given a priority before fixing.

## Method

For each Figma state, I capture the equivalent app screen (in `docs/audit/`) and compare:
- Copy / text
- Visual hierarchy & layout
- Tooltip presence and content
- State-machine behavior
- Component placement

Findings tagged: **🔴 BUG** (functional regression), **🟡 COPY** (text doesn't match), **🟠 LAYOUT** (placement wrong), **🔵 MISSING** (Figma has it, we don't), **🟢 OK** (matches).

### 2. Underlying Request drawer (gated explainer)
Figma: [`07-underlying-request.png`](./figma/07-underlying-request.png) | Current: [`audit/02-drawer-error-current.png`](./audit/02-drawer-error-current.png)

- 🔴 **BUG**: 401 error shows `Error Explainer · Feature Not Enabled`. **User requirement: errors should always have the explainer available, not gated.** Mock data needs `hasAiExplainer = true` for all `status >= 400`.
- 🟠 **LAYOUT**: Gated Explainer row is rendered *outside* the Response accordion. Figma shows it as the **third sub-row of Response** (after Headers and Body). When response.body is unavailable, the rows read: `Headers`, `Body — Not available`, `Error Explainer · Feature Not Enabled — via Advanced Logs & AI Features ⓘ`.
- 🔵 **MISSING**: ⓘ info icon at the end of the "via Advanced Logs & AI Features" link with a tooltip explaining why it's gated.
- 🔵 **MISSING (Figma frame 07)**: Two-tier drawer header showing parent request (`GET CRM | List Employees · 200`) above the underlying request (`POST Underlying Request · 401`). Currently only single header.

### 3. AI Explainer toasts
Figma: [`10-ai-explainer-feedback.png`](./figma/10-ai-explainer-feedback.png) (old)

Only the explicit Submit action should fire a toast (`"Feedback Submitted"`).

## Fix plan (in order)

1. **Mock data: error logs always have `hasAiExplainer: true`**. Eliminates the gated state on errors.
2. **Move Error Explainer into Response section** as a sub-row after Body. Section accepts any number of trailing sub-rows.
3. **Remove `"Rating Provided"` toast**. Submit alone fires `"Feedback Submitted"`.
4. **Add info ⓘ icon + tooltip** on the gated state (still kept for completeness even though unreachable for errors).
5. **Add `Logs ↗` link to the success toast** for parity with Figma.
6. **Two-tier drawer header** for Underlying Request views — deferred until we ship a dedicated "underlying request" route. Note in TODO.
